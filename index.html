Hi, I am Chen Kai

I am a passionate mobile application developer focused on building secure, transparent, and impactful digital solutions that address modern global challenges.

Target Alignment: SDG 10

The UN’s blueprint to reduce inequalities within and among countries, which drives my commitment to build transparent digital ledger frameworks to secure financial access for all segments of society.

Matric No: A207397
Programme: Software Engineering
Instructor: Dr. IZWAN
Academic Year: 2026

Lab Milestones

Lab 1: UI Core

Focused on the core elements of constructing declarative layouts natively utilizing Android's Jetpack Compose engine.

Reflection: Explored the fundamentals of declarative UI design using Jetpack Compose, mastering row, column, and basic state management.

Lab 2: Interaction

Delved into advanced click integrations, reactive state observation, stateful hoisting, and architectural state patterns.

Reflection: Implemented dynamic user interactions and state hosting, learning how to handle state updates gracefully within screen composables.

Lab 3: Material Design 3

Applied complex customization to application branding, styling elements, customized light/dark typography, and layout themes.

Reflection: Applied Material Design 3 guidelines to create a beautifully branded, cohesive user interface with customized color palettes and professional typographies.

Lab 4: Navigation Graph

Established a reliable multi-destination stack architecture inside Compose to route data seamlessly between layouts.

Reflection: Built a multi-screen app architecture using Jetpack Navigation, mastering route arguments and structured navigation graphs.

Lab 5: Room Storage

Engineered full local storage capabilities utilizing structural schema definitions, abstract DAOs, and asynchronous Coroutine flows.

Reflection: Integrated local persistence using Room Database, allowing users to safely store financial transactions on-device with offline accessibility.

Capstone Projects

Project 1: Wize Finance Base

Problem Statement: Lack of intuitive digital budget tracking and personal financial asset monitoring tools for general consumers to bridge information gaps.

Lessons Learned: Mastered state sharing across multiple screens using an overarching ViewModel and engineered clean UI flows with structured backstack management.
View on GitHub

Project 2: Wize Finance Connected Ecosystem

Problem Statement: Financial corruption and a lack of transparent audit tracking in institutional fund routing and public grant distribution.

Lessons Learned: Successfully unified four major Android technical pillars: Room for offline storage, Retrofit for real-time compliance validation, CameraX for hardware-level QR verification, and Firebase Firestore for a transparent public cloud ledger.
View on GitHub

Learning Journey Reflection

My learning journey from Lab 1 to Project 2 has been incredibly transformative. I evolved from designing static, single-screen layouts into architecting complex, production-ready, fully connected Android applications. Overcoming real-world integration hurdles, such as dependency alignment and asynchronous networking, deeply enhanced my problem-solving skills and solidified my confidence in modern Android engineering.

Skills & Tech Stack

Kotlin Jetpack Compose Room Database Retrofit REST API Firebase Firestore CameraX API MVVM Architecture State Management Git & GitHub

Resources & Engineering References

  • Retrofit 2 Library & Converter-GSON for remote network interfacing
  • Room Persistence Engine for reactive local relational storage mapping
  • CameraX Lifecycle API for abstracting hardware sensor camera preview streaming
  • Firebase Cloud Firestore Android SDK for multi-client public ledger distribution
  • Jetpack Jetnav Composable component infrastructure
Acknowledgement of AI Assistance: I explicitly acknowledge the valuable engineering assistance of AI tools in resolving gradle dependency version catalog blocks, sorting out structural asynchronous coroutine flow bottlenecks, and debugging compile-time namespace collisions during full-scale package re-architecting.
